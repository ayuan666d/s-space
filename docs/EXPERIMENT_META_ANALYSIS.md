## 三、🎯 底层核心逻辑：三大定律

通过对所有实验数据的交叉分析，我提取出**三条底层定律**，它们统一解释了所有成功和失败：

### 定律1：语义保护区定律

> **"注入的维度中，语义维度的扰动量与输出质量成反比。"**

> **注**：下文中的"彩票维度/彩票mask"是历史术语，现统一为"稀疏维度/选择维度"。根据轴认知理论，不存在"彩票轴"和"非彩票轴"的区别，只有"当前任务需要用哪些轴"的区别。

**证据链**：

| 实验 | 注入维度 | 结果 |
|------|---------|------|
| Route Cal v6 | 只推15稀疏选择维度（但忽略LM loss） | 1/20 💀 |
| Route Cal v7 | 联合训练，保持LM能力 | 16/20 🏆 |
| 20层注入 | 全1024维方向向量注入 | 修5破7 |
| SNav | 全1024维位移注入 | 修5破7 |
| SmartRouter v7 zero模式 | 0维度注入（不注入） | 11/14=79% ✅ |

**核心洞察**：v6的灾难性失败看似是"纯HS对齐"的问题，但更深层的原因是**纯HS对齐破坏了LM能力**——而这恰好等价于"语义维度被扰动后无法恢复"。v7加入0.3×LM_loss本质上是在**保护语义维度**。

**推论**：理想注入应只在稀疏选择维度（历史称"彩票维度"，15维）上施加位移，保持1009个语义维度不动。这样既切换了模式，又不扰动语义内容。

### 定律2：层次分工定律

> **"不同层有不同功能角色，等力均匀注入是最差策略。"**

**证据链**：

COS测量数据揭示的关键模式：

```
层间判别力排序（标准差从大到小）：
  L19: std=0.280 ← 判别力最强！因果vs叙事的核心分离层
  L11: std=0.084 ← 中等判别力
  L23: std=0.072 ← 输出门控
  L3:  std=0.053 ← 类型识别
  L15: std=0.051 ← 结构化程度
  L7:  std=0.044 ← 叙事瓶颈（判别力弱但轨迹停滞）
```

**L19的核心发现**：
- 推理类L19均值 = **-0.041**（偏向反推理方向）
- 叙事类L19均值 = **+0.033**（偏向叙事方向）
- L19方差是其他层的**3-6倍**
- **极端值**：因果推理最强偏移+0.501，合成谬误最强偏移-0.453

**这告诉我们**：L19是推理vs叙事的**核心决策层**，注入力度应该最大；而L7是**轨迹停滞层**，需要的不是方向性注入而是打破停滞。

**当前工程**：CORE=[3,7,11,15,19,23]等间距——完全没利用层次分工信息。

### 定律3：干预强度倒U定律

> **"存在最优干预强度，过弱无效，过强破坏，呈倒U型曲线。"**

**证据链**：

| 实验 | 干预强度 | 效果 |
|------|---------|------|
| Baseline | 0 | ~11/20 (55%) |
| embed_scale 0.5 | 弱 | 无改善 |
| embed_scale 1.0 | 等 | 无改善 |
| embed_scale 3.0 | 中 | 略有改善 |
| embed_scale 5.0 | 强 | 无改善 |
| subspace_gain 2x | 弱 | 无改善 |
| subspace_gain 5x | 中 | 略有改善 |
| subspace_gain 10x | 强 | 模型崩坏 |
| Route Cal v6 | 过强（纯HS优化） | 1/20 💀 |
| Route Cal v7 | 适中（联合训练） | 16/20 🏆 |
| SmartRouter COS | 自适应 | 15/20 |
| 20层均匀注入 | 过强 | 修5破7 |

**关键发现**：SmartRouter v7的"zero模式"（不注入）在数学/逻辑题上效果最好(11/14=79%)。这等价于**干预强度=0**在某些题型上是最优点。

**推论**：最优干预强度应该是**题型相关的**——数学题强度=0，叙事题强度较大，知识题中等。这恰好验证了倒U定律：不同题型有不同的最优干预点。

## 四、为什么"修5破7"？— 机制解释

20层注入和SNav都出现了"修5破7"的现象，这不是偶然的。用三大定律可以完整解释：

```
原来错误的题（5个）:
  → 模式不对 → 注入把模式推对了 → 修好了 ✅
  → 但同时扰动了语义维度 → 输出质量略有下降
  
原来正确的题（7个）:
  → 模式和语义都对
  → 注入把模式推偏了 → 模式失配
  → 同时扰动了语义维度 → 语义失真
  → 双重打击 → 输出崩溃，产生循环 💀
  
净效果: +5 -7 = -2
```

**根本原因**：全1024维注入违反了定律1（语义保护区），均匀等力注入违反了定律2（层次分工），强度过大违反了定律3（倒U曲线）。

## 五、SmartRouter v7 为什么有效？— 机制解释

SmartRouter v7的75%是目前运行时最优成绩，它恰好利用了三大定律的正面效应：

| 组件 | 利用的定律 | 机制 |
|------|-----------|------|
| **zero模式**（数学/逻辑不注入） | 定律3 | 干预强度=0是最优点 |
| **auto模式**（COS自省） | 定律3 | 自适应寻找倒U曲线峰值 |
| **minus模式**（叙事反向注入） | 定律2 | 叙事题需要释放创造力 |
| **plus模式**（知识正向注入） | 定律2 | 知识题需要结构化方向 |
| **类型特异方向**（type_dirs.pt） | 定律2 | 不同题型不同层需要不同方向 |
| **稀疏层选择**（top-6活跃层） | 定律1 | 减少注入的维度扰动 |
| **COS目标区间**[-0.15,-0.02] | 定律3 | 避免过度干预 |

**但SmartRouter仍然的问题**：
1. 注入的是**全1024维方向向量**，违反定律1
2. 层选择是**top-6活跃层**，不是**按层次分工**选，部分违反定律2
3. COS目标区间是**经验值**，没有理论支撑

## 七、验证与预测

基于三大定律，我做出以下**可证伪的预测**：

### 预测1：纯稀疏维度注入将消除"破7"现象

如果只推15维不动1009维，那么"修5破7"应该变成"修5破0"——因为语义维度不再被扰动。

**验证方法**：修改`_gen()`方法，注入向量只在15个稀疏选择维度上有值。

### 预测2：L19重点注入将提升推理类题目表现

当前CORE=[3,7,11,15,19,23]等间距，改为L19强度2-3倍于其他层。

**验证方法**：对因果推理/逻辑类题目，L19注入强度设为0.3，其他层0.1。

### 预测3：数学题不注入就是最优解

SmartRouter的zero模式已经验证了这一点(11/14=79%)。但zero模式的79%可能还不是上限——如果减少COS自省的尝试次数（避免在auto模式误判时过度注入），可能会更高。

### 预测4：Route Cal v7的16/20在推理时也可复现

v7的高分依赖于训练时优化。如果我们将v7的"联合训练"思想转化为推理时的"联合评估"——同时检查COS和输出语义质量——应该能进一步提升SmartRouter。

## 九、结论

### 为什么做了近百场实验却找不到确定有效的方案？

**答案**：因为我们在三个维度上同时犯错，而且这三个错误互相掩盖——

1. **维度错误**：全维注入破坏语义，导致"修5破7"
2. **层次错误**：均匀注入忽视分工，导致L19/L7关键层没被充分利用
3. **强度错误**：固定强度忽视题型差异，导致数学题被过注入

这三个错误**互相纠缠**：当你调高强度修复推理题，语义被扰动导致知识题崩坏；当你减少层数降低扰动，关键层又没被注入。结果就是**怎么调都不对**。

### 出路

三大定律告诉我们，出路不是"更精细地调参"，而是**从根本上改变注入的维度、层次和强度策略**：

1. 只推15维（定律1，稀疏选择维度）→ 消除语义扰动
2. 按层次分工分配（定律2）→ 充分利用L19/L7
3. 按题型自适应（定律3）→ 找到倒U曲线峰值

**最关键的一步**：实现纯稀疏维度注入。代码改动量极小，理论支撑极强，预期效果极大。
